# Bosmax Command Centre Agent Rules

- Treat every file under `knowledge-pack/` as a production knowledge-pack asset.
- No unsolicited rewrite of copywriting, prompts, dialogue, CTA text, product truth, avatar descriptions, workbook content, or approved claims.
- Product truth outranks avatar style, scene styling, and persuasive copy.
- YAML, JSON, CSV, Markdown, TXT, and XLSX assets must remain parseable or openable.
- Every change must be made through branch plus PR. Never push directly to `main` or `master`.
- Every PR must include validation proof, git proof, and an explicit status for known gaps.
- Do not claim `DONE` without commit SHA, PR URL, and validation evidence.
- Do not modify approved commercial copy without explicit operator approval.
- Report outcomes using `VERIFIED`, `NOT VERIFIED`, and `REMAINING GAPS`.
- If a YAML syntax repair is required, keep it syntax-only and document the exact quoted lines.
